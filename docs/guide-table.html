Advanced Table Editing ​

Tables are very versatile layout "building blocks". Liii STEM can not only create ordinary text tables but also precisely control the attributes of each cell; more broadly speaking, matrices, aligned formula lists, etc., are also typeset in the form of "tables".

For methods of table insertion and referencing, see Referencing Numbered Content.

This guide will help you further master table usage methods and provide detailed introductions to matrices and piecewise functions.

1 Tables ​

1.1 Types of Tables ​

  • Regular tables are conventional tables in the main text, without numbering or captions. They are classified based on width, centering, presence of labels, and borders.

  • Numbered tables have numbering and captions. Large tables occupy a single row, while small tables can be arranged side by side.

1.2 Basic Table Editing ​

  • Insert Table: Click the six-grid style button in the Mode Toolbar (as shown below) and select the table type.

  • Add/Delete Rows and Columns: Newly created tables are generally 1×1 empty tables. After selecting a table, the Environment Toolbar will display table-related operation sections.

The first four icons from left to right in the above image are: insert row above, insert column to the left, insert column to the right, insert row below; the last four are the corresponding deletion operations.

Tips: Liii STEM has Magic Paste functionality that allows you to directly paste tables from AI Large Language Model responses. This greatly saves you time creating empty tables + entering content, generating with one click, efficient and convenient!

1.3 Modifying Table Properties ​

  • In the Menu Bar, click Format -> Table to open the Table Properties
  • Or select the table and right-click to bring up the Function Bar

1.3.1 Setting Row and Column Count ​

You can set the number of rows and columns in the following three ways:

  • Click buttons in the Environment Toolbar
  • Set "Rows" and "Columns" in the Table Properties window
  • Right-click to bring up the Function Bar and set in Table -> Size

Additionally, you can declare minimum/maximum row and column counts for tables, making it easier to create custom table macros.

Note: If the set value for row/column attributes exceeds their allowed minimum and maximum range, the setting will not take effect, and the corresponding row/column attributes of the table will remain unchanged.

1.3.3 Page Splitting ​

For long tables that span across pages, you can enable page splitting by clicking Format -> Table -> Table Properties -> Allow Page Break in the Menu Bar.

Note: If cells have borders set, it is recommended to set the line/paragraph spacing of that paragraph to 0 to avoid extra white space at page breaks.

2 Common Mathematical Applications of Tables ​

Matrices are presented in Liii STEM as special forms of tables: you can add/delete rows and columns, set alignment and dimensions like editing tables, and then choose the outer delimiters (parentheses, square brackets, vertical lines, etc.).

Therefore, the vast majority of table tool editing methods also apply to matrices, as detailed below:

2.1 Matrices ​

In the Menu Bar, sequentially click Insert -> Math -> Displayed/Inline Formula.

Displayed formulas occupy an entire row, while inline formulas are embedded in the text.

In the formula environment, sequentially click Insert -> Table -> Matrix in the Menu Bar. This way, we generate a 1x1 empty matrix in the formula row, with other methods being the same as for tables.

2.2 Piecewise Functions ​

Piecewise functions are essentially two-column tables with a large brace on the left side (left column for expressions, right column for conditions), so they can be handled using table editing methods. Specifically:

  • In the Menu Bar, sequentially click Insert -> Math -> Displayed/Inline Formula
  • Manually input the function name, equal sign, and other parts.
  • Use the command \choice to insert the large brace and single-column table

    Small tip: You can quickly increase the number of rows by pressing Enter! The operations for adjusting rows and columns are the same as for tables.

  • Input expressions and other information you need to enter.
